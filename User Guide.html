v. 1.1.1

Streampool is a multi-tenant, easy scalable, fault tolerant content delivery service for distributing adaptive media streams to the end users. 

  • Supported formats: HLS, DASH, CMAF
  • Ingest for Live streams, Events and VOD assets
  • Just in time VOD packager
  • Content can be protected by ACL, Token and restricted by country 
  • Configurable DVR allows to have catch-up and timeshift features out of the box 
  • Extended online statistic for ingest and delivery
  • DRM Passthrough

1. Spots

Overview: Spot is an independent Streampool cluster consisting of one or several nodes located in one data center. One provider can have several spots in different locations (DC), as well as one spot can serve several providers. End users can be stitched to appropriate Spot based on flexible routing rules. 

The Main screen of spots monitoring can be displayed in several variants:

  • - Click to see Spots in Cards view.

  • - Click to see Spots in Table view.

  • - Click to see Spots in Map view.

  • Search field - Search Spots by Name.

  • Customers - Select Customers from the dropdown list to see Spots attached to them.

  • Filter by status - Select option to filter spots:

    • Total - Select to display all spots;
    • Online - Select to display Online spots only;
    • Offline - Select to display Offline spots only

  • Live statistics - Switch ON to see Spots statistics in real time. When Live statistics is disabled Spots information is updated after refreshing the page.

  • Add new spot - Click to open the for new spot creation.

    • Name - Type a name for the spot;
    • API Key - Insert the API key.
    • Server URL - Insert the Server URL.
    • Location - Fill the fields with information about Server location.
      • - Click to fill Location fields automatically based on the Server URL.
      • Country - Type the name of the country the Server is located in.
      • City - Type the name of the city the Server is located in.
      • 'Latitude - Insert the Latitude of the Server location.
      • Longitude - Insert the longitude of the Server location.

1.2.3 Pools

**Overview: Pool is a group of streams with unique settings for content source and delivery. 

1.2.3.1 Pool Details

Path: Spots - selected Spot - Pools - Add new pool/ Edit selected pool

  • Name - Type a unique name for the Pool.
A name of the Pool should not contain spaces and special characters and can’t be modified after it is saved. 
A name of the pool must be unique as it is a part of Stream URI.

  • Customer - Select a customer for this Pool.

  • Content source - Select the type of pool scheme.

    • Ingest - Content is uploading to the Streampool (PUSH scheme):
      - Content type - Select the type of the content to upload to Streampool.
      - Live - Live content stored depending on DVR settings:
      - DVR - Enable to allow viewers to pause, rewind, and continue during the live content.
      - Time of content storing - Type the number and select the unit of time.
      - - Click to increase/decrease number of time units.
      - - Click to delete the input.
      - Vod - Vod content is always stored.
      - Storage - Select the storage to save content to.
      - Security - Select a type of permission to upload content:
      - Allow all;
      - Allow from following IPs/CIDR Blocks
      - Advanced Settings - Click to see Advanced settings options.
      - Bunch size - Type a size for the bunch in bytes.
      - - Click to increase/decrease number of bytes.
      - - Click to delete the input.
      - DVR Masks - Type the DVR mask.

    • Origin (pull) - Streampool pulls content from the Origin server (PULL scheme). Configuring settings of Origin (Pull) type of content result in a link for the Origin request
      - Protocol - Select the Protocol.
      Available values:
      - Same as request - requires HTTP Port and HTTPS Port settings;
      - HTTP - requires HTTP Port Settings;
      - HTTPS - requires HTTPS Port Settings.
      - Host name - Type the Host name.
      - Host Header.
      Available values:
      - Incoming Hostname;
      - Origin Hostname;
      - Custom - Type a Custom Header.
      - Path - Select the Path.
      Available values:
      - Keep incoming path
      - Use prefix
      - Path Prefix - Type Path Prefix.
      - Replace Path
      - New path - Type New Path.

    • VOD Packager - media content (mp4 files) will be packaged on demand (just in time) into required format (HLS, DASH).

      • Packager config - Select config for the packager.
A special configuration is required to provide access to the “Packagers(insert link)” section. To get access contact our support team.  

  • Delivery -  this section is intended for content protection settings. It makes delivery content accessible to viewers and prevent unauthorized views.

    • Token Authentication - Switch ON to protect content by a token.

    • Token - Specify a token or click the “Generate Key” button to generate it automatically.

      • - Click to copy Token Key.

    • Geo Protection - Switch ON to protect content by imposing restrictions on geographical locations to get access from.

      • Mode - select the type of rule to impose.
        Available values:
        • Allow from following locations
        • Deny from following locations
      • Countries - Select one or several countries to impose the rule on.
        • - Click to delete the Country from the list.
      • Whitelist - Add IP/CIDR to give access to content to.
        • - Click to delete the IP/CIDR from the list.

    • IP/CIDR - Switch ON to specify IP/CIDR to impose rule on.

      • Mode.
        Available values:

        • Allow from following IPs/CIDR Blocks;
        • Deny from following IPs/CIDR Blocks.

      • Add IP/CIDR - Click to open pop-up for adding IP/CIDR. 

        - IP/CIDR # - Specify IP/CIDR;
- Add more - Click tp add one more IP/CIDR.
- Add IP - Click to add specified IPs/CIDRs to the settings.

      • - Click to delete IP/CIDR from the list.

      • - Click to delete all IPs/CIDRs from the list.

1.2.5 Packagers

Overview: Delivering content for multiple devices using HTTP streaming can be done via HLS/DASH/CMAF streaming formats. The Streampool can package the content “on the fly” or “just in time” based on clients’ requests.
A special option is required to provide access to the “Packagers” option. To get access contact our support team.
